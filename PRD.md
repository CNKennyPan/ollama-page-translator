# Ollama 本地翻译 Chrome 插件 — PRD

## 1. 产品概述

基于用户本地运行的 Ollama 服务，在 Chrome 浏览器中实现页面文本的翻译和替换。所有翻译请求通过本地 API 完成，数据不出本机。

---

## 2. 核心工作流

```
用户打开网页 → 点击插件图标/自动触发 → 内容脚本提取页面文本节点
  → 发送到 ollama (/api/chat 或 /api/generate) → 接收翻译结果
  → 逐一替换页面上的原文 → 缓存结果避免重复请求
```

## 3. 功能选项（需决策）

### 3.1 触发方式
| 选项 | 说明 | 建议 |
|------|------|------|
| A. 手动点击图标翻译 | 用户点击 toolbar icon 后翻译整个页面 | MVP |
| B. 右键菜单翻译选中文本 | 选中一段文字，右键翻译并替换 | MVP |
| C. 自动翻译 | 页面加载后自动翻译（可按域名白名单） | V2 |
| D. 划词即译（不替换，弹出浮窗） | 选中即显示翻译浮窗 | V2 |

### 3.2 翻译范围
| 选项 | 说明 | 建议 |
|------|------|------|
| A. 全文翻译（所有可见文本节点） | 替换页面全部文字 | MVP |
| B. 仅翻译选区 | 只翻译用户选中的文本 | MVP |
| C. 翻译特定元素（如 main/article） | 跳过导航、广告等 | V2 |
| D. 按段落/句子逐段翻译 | 保留原文结构，逐段替换 | V2 |

### 3.3 语言策略
| 选项 | 说明 | 建议 |
|------|------|------|
| A. 用户指定源语言和目标语言 | 手动选择 from→to | MVP |
| B. 自动检测源语言 | 让 ollama 或库检测源语言 | V2 |
| C. 双语对照（原文 + 译文） | 保留原文，译文跟在后面 | V2 |
| D. 仅目标语言（替换原文） | 直接替换页面文字 | MVP |

### 3.4 Ollama 模型选择
| 选项 | 说明 | 建议 |
|------|------|------|
| A. 使用大模型（llama3.1 70B, qwen2.5:32b）| 翻译质量高，速度慢 | - |
| B. 使用中型模型（qwen2.5:14b, llama3.1:8b）| 折中 | MVP(可选) |
| C. 使用小型模型（qwen2.5:7b, phi4）| 速度快，质量尚可 | MVP(推荐) |
| D. 专用翻译模型（如 madlad400, nllb）| 专为翻译优化 | 高级 |

**用户需在 Ollama 中自行 pull 模型。插件提供下拉菜单选择已安装的模型。**

### 3.5 内容替换策略
| 选项 | 说明 | 建议 |
|------|------|------|
| A. 逐个文本节点直接替换 textContent | 简单直接，丢失样式 | MVP |
| B. 创建翻译标注（高亮译文） | 添加视觉效果标记已翻译 | V2 |
| C. 保留原文，hover 显示译文 | 不替换，交互式查看 | V2 |
| D. 节点克隆 + 替换（保留结构） | 更好的 DOM 保留 | MVP |

### 3.6 动态内容处理
| 选项 | 说明 | 建议 |
|------|------|------|
| A. 一次翻译，不做后续处理 | 页面变化不重新翻译 | MVP |
| B. MutationObserver 监听 DOM 变化 | 新出现的内容自动翻译 | V2 |
| C. 只翻译初始加载内容，忽略动态 | 简单可靠 | MVP |

### 3.7 需要跳过的元素
| 选项 | 说明 |
|------|------|
| `<script>`, `<style>`, `<svg>`, `<canvas>` | 必须跳过 |
| `<code>`, `<pre>`, 代码块 | 建议跳过 |
| 已翻译过的节点（data 属性标记） | 防重复翻译 |
| 非常短的文本（< 2 字符） | 跳过 |
| URL、邮箱、数字 | 建议保留原文 |

### 3.8 缓存策略
| 选项 | 说明 | 建议 |
|------|------|------|
| A. 内存缓存（页面级） | 当前页面不重复翻译 | MVP |
| B. localStorage/IndexedDB 持久缓存 | 跨页面/跨 session | V2 |
| C. 按域名 + 原文 hash 缓存 | 复用已翻译结果 | V2 |
| D. 不缓存 | 每次都请求 | 不推荐 |

### 3.9 错误处理
| 选项 | 说明 |
|------|------|
| Ollama 未运行 | 提示用户启动 |
| 网络超时 | 重试机制 |
| 模型加载中 | 等待或提示 |
| 部分节点翻译失败 | 跳过失败的节点，继续其他 |
| 请求被限流 | 队列控制，并发数限制 |

### 3.10 UI / 交互
| 组件 | 说明 | MVP |
|------|------|------|
| Toolbar 图标 | 点击弹出 popup | ✅ |
| Popup 面板 | 语言选择、翻译按钮、状态 | ✅ |
| 语言选择下拉 | 源语言 / 目标语言 | ✅ |
| 翻译进度指示 | 已翻译 x/y 个节点 | MVP(简单) |
| 还原原文按钮 | 一键恢复 | MVP |
| 设置页面（options） | 模型选择、Ollama 地址、跳过规则 | ✅ |
| 浮动翻译球 | 页面内悬浮控制 | V2 |
| 快捷键 | Alt+T 翻译，Alt+R 还原 | V2 |

### 3.11 权限需求
| 权限 | 用途 |
|------|------|
| `activeTab` | 访问当前页面 DOM |
| `storage` | 保存设置 |
| `contextMenus` | 右键菜单 |
| `host_permissions` | 连接本地 `http://127.0.0.1:11434`（可选具体域名）|

### 3.12 隐私与安全
- 所有请求发往 `localhost:11434`，数据不离开本机
- 不收集用户数据，不上传任何信息
- 不开额外网络权限，只连本地 Ollama
- 开源，代码透明

## 4. 技术架构

```
┌─────────────────────────────────────┐
│  Chrome Extension (Manifest V3)     │
│                                     │
│  ┌───────────┐  ┌────────────────┐  │
│  │ Popup UI  │  │ Options Page   │  │
│  └─────┬─────┘  └───────┬────────┘  │
│        │                 │           │
│  ┌─────▼─────────────────▼────────┐  │
│  │   Background Service Worker    │  │
│  │   - 管理翻译队列              │  │
│  │   - 调用 Ollama API           │  │
│  │   - 缓存管理                  │  │
│  └───────────────┬───────────────┘  │
│                  │                   │
│  ┌───────────────▼───────────────┐  │
│  │   Content Script              │  │
│  │   - 提取 DOM 文本节点         │  │
│  │   - 接收翻译结果替换 DOM      │  │
│  │   - 标记已翻译节点            │  │
│  └───────────────┬───────────────┘  │
│                  │                   │
└──────────────────┼──────────────────┘
                   │
                   │ HTTP (localhost:11434)
                   │
┌──────────────────▼──────────────────┐
│  Ollama (本地运行)                  │
│  /api/chat 或 /api/generate         │
│  模型: qwen2.5 / llama3 / phi4 等   │
└─────────────────────────────────────┘
```

### 组件通信

```
Popup → chrome.runtime.sendMessage → Service Worker
Content Script → chrome.runtime.sendMessage → Service Worker
Service Worker → fetch → http://localhost:11434/api/chat
Service Worker → chrome.tabs.sendMessage → Content Script
```

## 5. MVP 范围定义

### MVP 包含
- [x] ManualV3 项目结构
- [x] Toolbar 图标 + Popup（语言选择 + 翻译/还原按钮）
- [x] 全文翻译（替换文本节点，跳过 script/style 等）
- [x] 用户指定源语言和目标语言
- [x] 连接到本地 Ollama (`http://127.0.0.1:11434`)
- [x] Ollama 模型选择（options 页面）
- [x] 内存级缓存（页面内不重复翻译）
- [x] 还原原文功能
- [x] 右键菜单翻译选中文本
- [x] 错误提示（Ollama 未运行等）
- [x] 翻译进度提示
- [x] Ollama 地址可配置

### MVP 不包含
- [ ] 自动语言检测
- [ ] MutationObserver 动态内容翻译
- [ ] 双语对照模式
- [ ] 持久化缓存 (IndexedDB)
- [ ] 浮动翻译球
- [ ] Shadow DOM 穿透翻译
- [ ] 划词浮窗翻译
- [ ] 自动翻译（按域名白名单）
- [ ] 快捷键
- [ ] 翻译历史记录
- [ ] iframe 内翻译

## 6. 待决策项（请选择）

| # | 问题 | 选项 | 我的建议 |
|---|------|------|----------|
| 1 | 默认用哪个模型？ | qwen2.5:7b / llama3.2:3b / phi4 / 其他 | qwen2.5:7b（中文好，体积适中）|
